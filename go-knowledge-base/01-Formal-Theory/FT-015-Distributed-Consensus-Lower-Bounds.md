│                      FLP Impossibility Result                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  定理 (Fischer, Lynch, Paterson, 1985):                                      │
│                                                                              │
│  在异步系统中，即使只有一个进程可能故障，也不存在确定性的共识算法。              │
│                                                                              │
│  "No deterministic consensus algorithm exists for asynchronous systems      │

│                                                                              │
│  异步系统假设:                                                                │
│  - 消息延迟无界 (unbounded message delay)                                     │

│                                                                              │

│                                                                              │
│  核心矛盾:                                                                    │
│  ┌─────────────┐      ┌─────────────┐                                       │
│  │  安全要求   │      │  活性要求   │                                       │

│  │             │      │             │                                       │
│  │  一致性     │      │  终止性     │                                       │

│  └─────────────┘      └─────────────┘                                       │
│         │                    │                                               │
│         └────────┬───────────┘                                               │
│                  │                                                           │

│         在异步系统中不可同时满足                                              │
│                                                                              │

## FLP 证明概述

### 双价初始配置

```
定义:
- 0-价配置: 所有可达决定都是 0
- 1-价配置: 所有可达决定都是 1
- 双价配置: 既可到达 0-决定，也可到达 1-决定

引理 1: 存在双价初始配置
证明:
考虑两个初始配置 C0 和 C1，仅在进程 p 的输入值不同
假设所有初始配置都是单价，则存在相邻配置从 0-价变为 1-价
在这两个配置之间运行 p 崩溃的场景，其他进程无法区分
导致矛盾。

图示:

C0 (p=0, q=0) ──► 0-价
    │
    │ p 改变输入
    ▼
C1 (p=1, q=0) ──► 1-价

如果存在双价配置，则算法可能永不终止。
```

### 异步调度导致不可终止

```
引理 2: 从双价配置出发，存在无限执行序列始终不决定

构造方法:
1. 从双价配置 C 开始
2. 选择事件 e = (p, m)，使得:
   - C 中应用 e 到达的配置仍是双价的
   - 这样的 e 总是存在 (除非 C 已是单价)
3. 延迟 e 足够长时间
4. 重复步骤 2-3

结果: 无限执行序列，永不决定
```

## CAP 定理

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      CAP Theorem                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  定理 (Brewer, 2000):                                                        │
│  分布式数据系统不可能同时满足以下三个性质:                                     │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                                                                     │    │
│  │     C          A           P                                        │    │
│  │  Consistency  Availability  Partition Tolerance                     │    │
│  │  (一致性)     (可用性)      (分区容错性)                             │    │
│  │                                                                     │    │
│  │     ◄──────── 只能选两个 ────────►                                  │    │
│  │                                                                     │    │
│  │  常见组合:                                                           │    │
│  │  • CA: 传统数据库 (单节点或同步复制)                                  │    │
│  │  • CP: HBase, MongoDB, Redis Cluster, etcd, ZooKeeper               │    │
│  │  • AP: Cassandra, DynamoDB, Couchbase, Riak                         │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  注意: 网络分区必然发生，所以实际上在 C 和 A 之间选择                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 其他下界结果

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Other Lower Bounds                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. 消息复杂度下界                                                           │
│     - 异步共识至少需要 O(n²) 条消息                                          │
│     - Dolev, Reischuk (1985)                                                 │
│                                                                              │
│  2. 故障容忍下界                                                             │
│     - 崩溃容错: 需要 2f+1 个节点 (多数派)                                     │
│     - 拜占庭容错: 需要 3f+1 个节点                                            │
│     - Pease, Shostak, Lamport (1980)                                         │
│                                                                              │
│  3. 轮复杂度下界                                                             │
│     - 确定性共识至少需要 f+1 轮                                              │
│     - Fischer, Lynch (1982)                                                  │
│                                                                              │
│  4. 时间复杂度 (同步系统)                                                    │
│     - 已知最优: O(1) 轮 (Paxos/Raft)                                         │
│     - 下限: Ω(1) 显然                                                        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```
