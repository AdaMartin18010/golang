# 子类型关系 (Subtyping)

> **分类**: 形式理论
> **难度**: 进阶

---

## 定义

子类型关系 `<:` 定义为：

```
t₁ <: t₂  ⟺  t₁ 的实例可以在任何需要 t₂ 的上下文中使用

Liskov 替换原则: 子类型必须能替换父类型
```

---

## Go 的子类型规则

### 结构子类型

```
t_S <: t_I  ⟺  methods(t_I) ⊆ methods(t_S)
```

### 接口子类型

```
t_I₁ <: t_I₂  ⟺  methods(t_I₂) ⊆ methods(t_I₁)

注意: 接口子类型是反变的
```

---

## 性质

### 自反性

```
t <: t
```

### 传递性

```
t₁ <: t₂, t₂ <: t₃  ⟹  t₁ <: t₃
```

### 反对称性（接口）

```
t_I₁ <: t_I₂, t_I₂ <: t_I₁  ⟹  methods(t_I₁) = methods(t_I₂)
```

---

## 空接口

```go
interface{}
```

所有类型都满足空接口：

```
∀t: t <: interface{}
```
