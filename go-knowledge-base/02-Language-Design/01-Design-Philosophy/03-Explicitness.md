# 显式优于隐式 (Explicit over Implicit)

> **分类**: 语言设计

---

## 核心原则

代码应该**表达意图**，而不是依赖隐藏的逻辑。

---

## 体现

### 1. 错误处理

```go
// Go: 显式错误检查
f, err := os.Open("file")
if err != nil {
    return err
}

// 对比 Java/Python: 异常隐藏调用栈
```

### 2. 导入

```go
// 显式导入
import "fmt"

// 未使用导入报错
```

### 3. 初始化

```go
// 显式初始化
var x int = 0
y := 0

// 无隐式零值依赖（但零值规则清晰）
```

---

## 对比隐式语言

| 特性 | Go (显式) | Python/Java (隐式) |
|------|-----------|-------------------|
| 错误处理 | if err != nil | try-catch 或隐式传播 |
| 方法重写 | 无继承 | 需标记 @Override |
| 类型转换 | 需显式转换 | 自动装箱拆箱 |
| 依赖注入 | 显式传递 | 框架自动注入 |

---

## 优点

- 代码易读
- 行为可预测
- 调试简单

## 缺点

- 代码冗长
- 需要更多输入
