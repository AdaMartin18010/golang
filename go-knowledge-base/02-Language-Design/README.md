## 核心设计原则

### 1. 简洁性

```go
// Go 的简洁: 用一种方式做一件事
if err != nil {
    return err
}

// 没有异常，没有 try-catch-finally
// 显式错误处理
```

### 2. 组合

```go
// 通过嵌入组合功能
type Reader struct { ... }
type Writer struct { ... }

type ReadWriter struct {
    Reader      // 嵌入
    Writer      // 嵌入
}
```

### 3. 并发

```go
// Goroutine: 轻量级线程
go function()

// Channel: 类型安全通信
ch := make(chan int)
ch <- value      // 发送
value := <-ch    // 接收
```

## 设计决策分析

### 为什么不要泛型 (早期)

1. **复杂性**: 增加语言复杂度
2. **编译时间**: 可能影响编译速度
3. **替代方案**: interface{} + 反射/代码生成

### 为什么后来加入泛型

1. **代码重复**: 没有泛型导致大量重复代码
2. **类型安全**: interface{} 失去类型检查
3. **性能**: 反射有运行时开销
4. **社区压力**: 最大特性请求

### 为什么使用 CSP

1. **简单**: 比共享内存+锁更容易理解
2. **安全**: 避免数据竞争
3. **可组合**: 容易构建复杂系统
4. **历史验证**: Erlang、Occam 成功应用
