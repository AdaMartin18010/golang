## 核心创新

从**对象级扫描**改为**Span级扫描**。

## 技术细节

### 内存布局

- 8 KiB 连续 span
- 每对象 2 bits 元数据 (Seen + Scanned)

### 算法变化

```
传统 GC: 对象级深度优先扫描
Green Tea: Span级广度优先扫描
```

### SIMD 优化

- AVX 向量指令
- 特别适合小对象 (≤512 bytes)

## 性能数据

| 指标 | 改善 |
|------|------|
| GC 开销 | -10~40% |
| Intel Ice Lake+ | 额外 -10% |
| Cache Misses | -50% |
| 小对象分配 | 最高 -69% |

## 权衡

### 优势

- 显著降低 GC 延迟
- 更好的缓存局部性

### 劣势

- RSS 增加 8-15%
- 某些场景可能不适用
