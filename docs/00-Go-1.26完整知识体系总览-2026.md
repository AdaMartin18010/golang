## 2. 🏗️ 语言层：语法、语义与形式化

### 2.1 语法基础

#### 2.1.1 new() 优化（Go 1.26 编译器改进）

> ⚠️ **重要纠正**: 之前文档中关于 "new() 表达式" 的描述是**错误的**。
> Go 1.26 **没有**改变 `new()` 的语法，`new(expression)` 不是有效的 Go 语法。
>
> **实际的改进**是编译器优化了逃逸分析，让更多 `new()` 调用可以在栈上分配。

**编译器优化**:

```go
// 以下代码在 Go 1.26 中会自动优化
func getPointer() *int {
    ptr := new(int)  // 可能栈分配（Go 1.26 优化）
    *ptr = 42
    return ptr  // 如果需要返回，仍会逃逸到堆
}
```

**语义说明**:

- `new(T)` 仍然是唯一语法，T 必须是类型，不能是表达式
- Go 1.26 编译器改进了逃逸分析算法
- 更多场景下 `new()` 分配的内存可以在栈上，提升性能

**实际应用场景**:

```go
// JSON 可选字段（传统做法，仍然有效）
type User struct {
    Name string
    Age  *int `json:"age,omitempty"`
}

// 传统方式创建指针
age := calculateAge(birthDate)
user := User{
    Name: "Alice",
    Age:  &age,  // 取地址（仍然是标准做法）
}
```

**性能收益**:

- 无需修改现有代码
- 编译器自动优化内存分配
- 栈分配比堆分配更快，减少 GC 压力

#### 2.1.2 泛型自引用类型（Go 1.26 重大特性）

**形式化定义**:

```
类型参数 T 的约束可以引用定义 T 的泛型类型本身
T Constraint[T] where Constraint 包含对 T 的引用
```

**语法实现**:

```go
// 自引用接口约束
type Adder[A Adder[A]] interface {
    Add(A) A
}

// 使用示例
type IntAdder struct { value int }

func (a IntAdder) Add(other IntAdder) IntAdder {
    return IntAdder{value: a.value + other.value}
}
```

**语义分析**:

- 解决递归类型约束问题
- 支持 F-bounded polymorphism
- 编译器通过延迟绑定解析自引用

**项目应用**:

```go
// internal/domain/interfaces/specification_go126.go
type SpecificationGo126[T any, S SpecificationGo126[T, S]] interface {
    IsSatisfiedBy(entity *T) bool
    And(other S) S
    Or(other S) S
    Not() S
}
```

### 2.3 错误处理语义

#### 2.3.1 errors.AsType 泛型实现（Go 1.26）

**类型签名**:

```go
func AsType[E error](err error) (E, bool)
```

**形式化语义**:

```
AsType : error → (E × bool)
AsType(err) = (e, true)  if ∃e: error, e : E ∧ errors.As(err, &e)
AsType(err) = (zero(E), false) otherwise
```

**与 errors.As 对比**:

| 特性 | errors.As | errors.AsType |
|------|-----------|---------------|
| 语法 | `var e *MyError; errors.As(err, &e)` | `e, ok := errors.AsType[*MyError](err)` |
| 类型安全 | 运行时检查 | 编译期检查 |
| 性能 | 反射开销 | 泛型优化，更快 |
| 易用性 | 需要预声明变量 | 直接返回 |

**项目应用示例**:

```go
// internal/infrastructure/database/ent/ent.go
// 传统写法
func IsValidationError(err error) bool {
    var e *ValidationError
    return errors.As(err, &e)
}

// Go 1.26 优化写法
func IsValidationErrorGo126(err error) bool {
    _, ok := errors.AsType[*ValidationError](err)
    return ok
}
```

### 3.2 日志设计模式

#### 3.2.1 MultiHandler 架构设计

**设计问题**: 如何同时输出到多个目标（控制台、文件、远程）？

**Go 1.26 解决方案**:

```go
// 多处理器模式
handlers := []slog.Handler{
    slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: slog.LevelDebug,
    }),
    slog.NewTextHandler(file, &slog.HandlerOptions{
        Level: slog.LevelWarn,
    }),
}

multiHandler := slog.NewMultiHandler(handlers...)
logger := slog.New(multiHandler)

// Enabled: 任一处理器启用即启用
// Handle: 调用所有启用处理器的 Handle
// WithAttrs/WithGroup: 分发到所有处理器
```

**架构图**:

```
┌─────────────────────────────────────┐
│         Application                 │
│    logger.Info("msg")               │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│      slog.NewMultiHandler           │
│  ┌─────────────────────────────┐    │
│  │   Enabled(ctx, level)       │    │
│  │   return any(handler.Enabled)│   │
│  └─────────────────────────────┘    │
└──────────────┬──────────────────────┘
               ↓
    ┌──────────┴──────────┐
    ↓                     ↓
┌──────────┐        ┌──────────┐
│ Handler1 │        │ Handler2 │
│  JSON    │        │  Text    │
│  stdout  │        │  file    │
└──────────┘        └──────────┘
```

**项目实现**:

```go
// pkg/logger/logger.go
func NewMultiOutputLogger(level slog.Level, handlers ...slog.Handler) *Logger {
    multiHandler := slog.NewMultiHandler(handlers...)
    return &Logger{Logger: slog.New(multiHandler), level: level}
}
```

## 5. 🆕 Go 1.26 新特性详解

### 5.1 语言特性

#### 5.1.1 泛型自引用类型详解

**理论背景**: F-bounded polymorphism

```go
// 数学定义：
// type C[X <: C[X]] 表示 C 的类型参数 X 必须是 C 的子类型

// Go 1.26 实现
type Comparable[C Comparable[C]] interface {
    Compare(other C) int
}

// 实现类必须满足：IntCmp <: Comparable[IntCmp]
type IntCmp struct { value int }

func (a IntCmp) Compare(other IntCmp) int {
    return a.value - other.value
}
```

**实际应用：表达式树**:

```go
// 使用自引用泛型构建类型安全的表达式树
type Expr[E Expr[E]] interface {
    Eval() float64
    String() string
    Add(other E) E
    Mul(other E) E
}

type Number struct { val float64 }

func (n Number) Eval() float64 { return n.val }
func (n Number) String() string { return fmt.Sprintf("%g", n.val) }
func (n Number) Add(other Number) Number { return Number{n.val + other.val} }
func (n Number) Mul(other Number) Number { return Number{n.val * other.val} }
```

#### 5.1.2 new() 优化说明（纠正）

> ⚠️ **重要纠正**: 之前关于 "new() 表达式" 的描述是**错误的**。

**正确语法**:

```
NewExpr = "new" "(" Type ")"
```

- `new()` 只接受**类型**，不接受表达式
- `new(int)` ✓ 合法
- `new(42)` ✗ 非法
- `new(int(42))` ✗ 非法

**Go 1.26 实际改进**:

1. **编译器优化**: 改进了逃逸分析算法
2. **栈分配**: 更多 `new()` 调用可以在栈上分配
3. **性能提升**: 无需代码改动，自动受益

**与 &T{} 对比**:

| 特性 | new(T) | &T{} |
|------|--------|------|
| 语法 | new(int) | x := 42; &x |
| 初始化 | 零值 | 可自定义 |
| Go 1.26 优化 | ✅ 栈分配更智能 | ✅ 栈分配更智能 |
| 使用场景 | 需要零值指针时 | 需要自定义初始值时 |

### 5.2 标准库增强

#### 5.2.1 errors.AsType

**实现原理**:

```go
// 编译器单态化生成的代码
func AsType[Err error](err error) (Err, bool) {
    var target Err
    if errors.As(err, &target) {
        return target, true
    }
    var zero Err
    return zero, false
}
```

**性能优势**:

- 避免运行时类型断言开销
- 编译期确定类型大小
- 栈分配目标变量

#### 5.2.2 slog.NewMultiHandler

**设计模式**: 组合模式 (Composite Pattern)

```go
type MultiHandler struct {
    handlers []Handler
}

func (m *MultiHandler) Enabled(ctx context.Context, level Level) bool {
    // 任一处理器启用即返回 true
    for _, h := range m.handlers {
        if h.Enabled(ctx, level) {
            return true
        }
    }
    return false
}

func (m *MultiHandler) Handle(ctx context.Context, r Record) error {
    // 调用所有启用的处理器
    for _, h := range m.handlers {
        if h.Enabled(ctx, r.Level) {
            if err := h.Handle(ctx, r); err != nil {
                return err
            }
        }
    }
    return nil
}
```
