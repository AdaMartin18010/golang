## 3. Web框架对比

### 主流框架对比

| 框架 | 性能 | 易用性 | 生态 | 学习曲线 | 适用场景 | 示例数量 |
|------|------|-------|------|---------|---------|---------|
| **net/http** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 低 | 简单API | 5个 |
| **Gin** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 低 | 通用Web | 10个 |
| **Echo** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 低 | 高性能API | 6个 |
| **Fiber** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | 低 | 极致性能 | 4个 |
| **Beego** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | 中 | 企业应用 | 3个 |
| **Iris** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ | 中 | 功能丰富 | 2个 |

### 框架选型建议

```go
// 场景1: 简单API（推荐 net/http）
func main() {
    http.HandleFunc("/", handler)
    http.ListenAndServe(":8080", nil)
}

// 场景2: RESTful API（推荐 Gin）
func main() {
    r := gin.Default()
    r.GET("/api/users", getUsers)
    r.Run(":8080")
}

// 场景3: 高性能API（推荐 Fiber）
func main() {
    app := fiber.New()
    app.Get("/api/users", getUsers)
    app.Listen(":8080")
}
```

**选型决策树**:
- **简单项目** → net/http
- **快速开发** → Gin
- **性能优先** → Fiber/Echo
- **企业项目** → Gin/Beego

## 6. 示例选型建议

### 按学习阶段

**初级（第1-2月）**:
```text
✅ Hello World → 变量类型 → 控制流 → 函数
✅ HTTP服务器 → RESTful API → 数据库CRUD
✅ Goroutine → Channel → 基础并发模式
```

**中级（第3-6月）**:
```text
✅ Gin框架 → JWT认证 → 中间件开发
✅ GORM → Redis缓存 → 数据库事务
✅ Worker Pool → Context → 复杂并发
✅ Docker部署 → CI/CD流程
```

**高级（第7-12月）**:
```text
✅ gRPC微服务 → 服务发现 → 链路追踪
✅ K8s部署 → Helm Charts → 监控系统
✅ 分布式事务 → 熔断降级 → API网关
✅ 完整电商平台 → 实时聊天系统
```

### 按职业方向

**后端开发**:
- 核心: Web框架 + 数据库 + 缓存
- 进阶: 微服务 + 消息队列 + 分布式
- 示例: 20-30个

**DevOps工程师**:
- 核心: Docker + K8s + CI/CD
- 进阶: 监控 + 日志 + 性能优化
- 示例: 15-20个

**架构师**:
- 核心: 设计模式 + 微服务架构
- 进阶: 分布式系统 + 高可用设计
- 示例: 全部示例
