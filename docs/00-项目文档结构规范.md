## 1. 📝 文档结构标准

### 1.1 标准文档结构

所有技术文档必须包含以下结构（按顺序）：

```markdown
# 文档标题

> **简介**: 一句话说明本文档的内容和目标读者

**版本**: v1.0
**更新日期**: YYYY-MM-DD
**适用于**: Go 1.25.3

---

### 1.1 二级章节

#### 1.1.1 三级章节（如需要）

## 📚 扩展阅读
```

### 1.2 必需章节

| 序号 | 章节名称 | Emoji | 说明 |
|------|---------|-------|------|
| 1 | 概述/理论 | 📚 | 概念、原理说明 |

| 3 | 最佳实践 | 🎯 | 推荐做法 |
| 4 | 扩展阅读 | 📚 | 相关资源 |

## 2. 🎯 章节编号规范

### 2.1 编号系统

使用**层级编号系统**，一级章节使用数字编号：

```markdown
## 1. 📚 一级章节
### 1.1 二级章节
#### 1.1.1 三级章节
```

### 2.2 编号规则

- ✅ **一级章节**: 使用数字编号（1, 2, 3...）
- ✅ **二级章节**: 使用数字编号（1.1, 1.2, 2.1...）
- ✅ **三级章节**: 使用数字编号（1.1.1, 1.1.2...）
- ❌ **不要**: 在一级标题使用多级编号

### 2.3 编号示例

✅ **正确示例**：

```markdown
## 1. 📚 理论分析
### 1.1 基本概念
#### 1.1.1 定义和特性
#### 1.1.2 工作原理

### 1.2 核心机制
#### 1.2.1 实现方式
```

❌ **错误示例**：

```markdown
## 📚 理论分析  # 缺少编号
### 基本概念  # 缺少编号
```

## 5. 📊 文档组织

### 5.1 文档目录结构

```text
docs/
├── architecture/          # 架构文档
│   ├── README.md         # 目录索引
│   ├── 00-知识图谱.md    # 导航文档（00-前缀）
│   ├── 00-概念定义体系.md
│   ├── 00-对比矩阵.md
│   └── *.md              # 具体文档
├── guides/               # 使用指南
│   └── *.md
└── ...
```

### 5.2 导航文档命名

导航文档使用 `00-` 前缀：

- `00-知识图谱.md` - 知识图谱
- `00-概念定义体系.md` - 概念定义
- `00-对比矩阵.md` - 对比矩阵

### 5.3 README 文档

每个目录应包含 `README.md`，提供：

- 目录概述
- 文档列表
- 快速导航
- 学习路径
