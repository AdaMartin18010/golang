## 1. Green Tea GC

### 1.1 概述

**Green Tea GC** 是 Go 的新一代垃圾回收器，在 Go 1.26 中默认启用。

| 指标 | 改进 |
|------|------|
| 延迟 | 10-40% 降低 |
| 吞吐量 | 提高 |
| 内存效率 | 更好 |
| 配置需求 | 无需配置 |

### 1.2 工作原理

```
Green Tea GC 架构:
────────────────────────────────────────

传统 GC                    Green Tea GC
─────────                  ────────────
Stop-the-world 标记    →   并发标记优化
全局写屏障              →   分代写屏障
固定堆增长比例          →   自适应堆大小

核心改进:
1. 并发标记: 减少 STW 时间
2. 写屏障优化: 降低 mutator 开销
3. 自适应策略: 根据应用行为调整
```

### 1.3 监控 GC 性能

```go
package main

import (
    "fmt"
    "runtime"
    "runtime/debug"
    "time"
)

func main() {
    // 1. 基本 GC 统计
    var m runtime.MemStats
    runtime.ReadMemStats(&m)

    fmt.Printf("=== GC 统计 ===\n")
    fmt.Printf("GC 次数: %d\n", m.NumGC)
    fmt.Printf("强制 GC 次数: %d\n", m.NumForcedGC)
    fmt.Printf("GC CPU 占比: %.2f%%\n", m.GCCPUFraction*100)

    // 2. GC 暂停时间分布
    fmt.Printf("\n=== GC 暂停时间 ===\n")
    for i := 0; i < 256 && i < int(m.NumGC); i++ {
        idx := (int(m.NumGC) + 255 - i) % 256
        pause := m.PauseNs[idx]
        if pause > 0 {
            fmt.Printf("  GC-%d: %d µs\n", m.NumGC-uint32(i), pause/1000)
        }
    }

    // 3. 内存分配统计
    fmt.Printf("\n=== 内存统计 ===\n")
    fmt.Printf("堆分配: %d MB\n", m.HeapAlloc/1024/1024)
    fmt.Printf("堆系统: %d MB\n", m.HeapSys/1024/1024)
    fmt.Printf("堆对象: %d\n", m.HeapObjects)
    fmt.Printf("下次 GC 目标: %d MB\n", m.NextGC/1024/1024)
}
```

### 1.5 性能对比

```go
package main

import (
    "fmt"
    "runtime"
    "testing"
    "time"
)

// 生成垃圾的基准测试
func BenchmarkGC(b *testing.B) {
    type Node struct {
        data [1024]byte  // 1KB
        next *Node
    }

    b.ReportAllocs()

    for i := 0; i < b.N; i++ {
        // 分配 100MB 临时数据
        var head *Node
        for j := 0; j < 100*1024; j++ {
            head = &Node{next: head}
        }
        runtime.KeepAlive(head)
    }
}

// 测量 GC 延迟
func measureGCLatency() {
    const sampleCount = 1000

    var maxPause time.Duration
    var totalPause time.Duration

    for i := 0; i < sampleCount; i++ {
        start := time.Now()
        runtime.GC()
        pause := time.Since(start)

        if pause > maxPause {
            maxPause = pause
        }
        totalPause += pause
    }

    fmt.Printf("GC 延迟: 平均=%v, 最大=%v\n",
        totalPause/sampleCount, maxPause)
}
```

## 2. cgo 开销优化

### 2.1 改进概述

Go 1.26 将 cgo 的基础开销减少了约 **30%**。

### 2.2 性能测试

```go
package main

/*
#include <stdlib.h>
#include <string.h>

void process(char* data, int len) {
    for (int i = 0; i < len; i++) {
        data[i] = data[i] + 1;
    }
}
*/
import "C"
import (
    "fmt"
    "time"
    "unsafe"
)

func cgoCall(data []byte) {
    if len(data) == 0 {
        return
    }
    cData := (*C.char)(unsafe.Pointer(&data[0]))
    C.process(cData, C.int(len(data)))
}

func pureGo(data []byte) {
    for i := range data {
        data[i] = data[i] + 1
    }
}

func main() {
    dataSize := 1024 * 1024 // 1MB
    iterations := 1000

    // 预热
    for i := 0; i < 100; i++ {
        data := make([]byte, dataSize)
        cgoCall(data)
    }

    // CGO 测试
    start := time.Now()
    for i := 0; i < iterations; i++ {
        data := make([]byte, dataSize)
        cgoCall(data)
    }
    cgoTime := time.Since(start)

    // Pure Go 测试
    start = time.Now()
    for i := 0; i < iterations; i++ {
        data := make([]byte, dataSize)
        pureGo(data)
    }
    goTime := time.Since(start)

    fmt.Printf("CGO 调用: %v\n", cgoTime)
    fmt.Printf("Pure Go:  %v\n", goTime)
    fmt.Printf("CGO 开销: %.2fx\n", float64(cgoTime)/float64(goTime))
}
```

### 2.3 最佳实践

```go
package main

// 1. 批量调用减少边界跨越
func processBatch(items [][]byte) {
    // 好的做法: 在 C 端处理批量
    C.process_batch((*C.Buffer)(unsafe.Pointer(&items[0])), C.int(len(items)))
}

// 2. 使用 sync.Pool 复用缓冲区
var bufferPool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 64*1024)
    },
}

func efficientCGO() {
    buf := bufferPool.Get().([]byte)
    defer bufferPool.Put(buf)

    // 使用 buf 进行 CGO 操作
}

// 3. 异步批处理
func asyncCGOProcessor(input <-chan []byte) {
    batch := make([][]byte, 0, 100)

    for data := range input {
        batch = append(batch, data)

        if len(batch) >= 100 {
            processBatch(batch)
            batch = batch[:0]
        }
    }

    if len(batch) > 0 {
        processBatch(batch)
    }
}
```

## 3. 编译器优化

### 3.1 切片栈分配优化

Go 1.26 编译器可以在更多情况下将切片的底层数组分配在栈上。

```go
package main

// 这种情况现在可以在栈上分配
calc := make([]int, 10)  // 小切片 -> 栈分配

// 优化场景:
// 1. 小容量切片 (通常 < 64KB)
// 2. 不逃逸的局部变量
// 3. 编译时可确定的容量

// 栈分配 vs 堆分配性能对比
func stackAlloc() {
    // 栈分配: ~10ns
    buf := make([]int, 100)
    _ = buf
}

func heapAlloc() []*int {
    // 堆分配: ~100ns + GC 压力
    buf := make([]*int, 100)
    return buf  // 逃逸到堆
}
```

### 3.2 逃逸分析改进

```go
package main

// Go 1.26 更好的逃逸分析

// 情况1: 不逃逸的切片字面量
func noEscapeSlice() int {
    nums := []int{1, 2, 3, 4, 5}  // 栈分配
    sum := 0
    for _, n := range nums {
        sum += n
    }
    return sum
}

// 情况2: 返回切片的切片（不修改）
func sliceOfSlices() [][]int {
    // 如果编译器能证明不修改，可能栈分配
    return [][]int{
        {1, 2, 3},
        {4, 5, 6},
    }
}

// 检查逃逸分析
go build -gcflags="-m" main.go
```


## 4. 标准库性能改进

### 4.1 io.ReadAll

```go
package main

import (
    "io"
    "strings"
    "testing"
)

// Go 1.26 io.ReadAll 性能提升约 2 倍
// 内存分配减少约 50%

func BenchmarkReadAll(b *testing.B) {
    data := strings.Repeat("x", 10*1024*1024) // 10MB

    b.ReportAllocs()
    b.ResetTimer()

    for i := 0; i < b.N; i++ {
        r := strings.NewReader(data)
        result, err := io.ReadAll(r)
        if err != nil {
            b.Fatal(err)
        }
        _ = result
    }
}

// 结果对比 (示意):
// Go 1.25:  ~50ms, ~30 allocations
// Go 1.26:  ~25ms, ~10 allocations
```

### 4.2 image/jpeg

```go
package main

import (
    "image/jpeg"
    "os"
    "testing"
)

// Go 1.26 全新的 JPEG 编解码器
// - 更快
// - 更准确
// - 但输出可能有略微不同

func BenchmarkJPEGEncode(b *testing.B) {
    // 加载测试图像
    f, _ := os.Open("test.jpg")
    defer f.Close()

    img, _ := jpeg.Decode(f)

    b.ReportAllocs()
    b.ResetTimer()

    for i := 0; i < b.N; i++ {
        out, _ := os.CreateTemp("", "bench-*.jpg")
        jpeg.Encode(out, img, &jpeg.Options{Quality: 85})
        out.Close()
    }
}
```

### 4.3 fmt.Errorf

```go
package main

import (
```
