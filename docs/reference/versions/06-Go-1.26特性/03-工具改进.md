## 1. go fix 完全重写

### 1.1 变化概述

Go 1.26 完全重写了 `go fix` 命令：

- **基于分析框架**: 使用与 `go vet` 相同的 Go 分析框架
- **Modernizers**: 自动更新代码以使用新特性
- **Inline 分析器**: 自动内联标记的函数
- **移除旧 fixers**: 所有历史 fixers 已被移除

### 1.2 基本使用

```bash
# 运行所有 modernizers
go fix ./...

# 预览变化 (不实际修改)
go fix -n ./...

# 指定特定分析器
go fix -fix=modernize ./...

# 应用特定修复
go fix -fix=slicescontains ./...
```

### 1.3 与旧版本对比

| 特性 | Go 1.25 及之前 | Go 1.26 |
|------|---------------|---------|
| 实现方式 | 自定义解析 | 基于分析框架 |
| 修复范围 | 语言版本迁移 | 现代化改进 |
| 扩展性 | 困难 | 容易 (与 vet 共享框架) |
| 分析能力 | 基础 | 深度语义分析 |

## 2. Modernizers 详解

### 2.1 什么是 Modernizers

Modernizers 是一系列代码分析器，自动将代码更新为使用：

- 新的语言特性
- 改进的标准库 API
- 更简洁的惯用写法

**保证**: Modernizers 不会改变程序行为。

### 2.2 内置 Modernizers 列表

```go
// ===== slices 包 Modernizers =====

// 1. contains - 使用 slices.Contains
// 修复前:
found := false
for _, v := range slice {
    if v == target {
        found = true
        break
    }
}

// 修复后:
found := slices.Contains(slice, target)


// 2. index - 使用 slices.Index
// 修复前:
idx := -1
for i, v := range slice {
    if v == target {
        idx = i
        break
    }
}

// 修复后:
idx := slices.Index(slice, target)


// 3. sort - 使用 slices.Sort
// 修复前:
sort.Slice(slice, func(i, j int) bool {
    return slice[i] < slice[j]
})

// 修复后:
slices.Sort(slice)


// ===== maps 包 Modernizers =====

// 4. mapsclone - 使用 maps.Clone
// 修复前:
clone := make(map[K]V, len(original))
for k, v := range original {
    clone[k] = v
}

// 修复后:
clone := maps.Clone(original)


// 5. mapscopy - 使用 maps.Copy
// 修复前:
for k, v := range src {
    dst[k] = v
}

// 修复后:
maps.Copy(dst, src)


// ===== strings 包 Modernizers =====

// 6. cutprefix - 使用 strings.CutPrefix
// 修复前:
if strings.HasPrefix(s, prefix) {
    s = s[len(prefix):]
}

// 修复后:
s, found := strings.CutPrefix(s, prefix)


// 7. cutsuffix - 使用 strings.CutSuffix
// 修复前:
if strings.HasSuffix(s, suffix) {
    s = s[:len(s)-len(suffix)]
}

// 修复后:
s, found := strings.CutSuffix(s, suffix)


// ===== context 包 Modernizers =====

// 8. withoutcancel - 使用 context.WithoutCancel
// 修复前:
ctx2, cancel := context.WithCancel(context.Background())
defer cancel()
ctx2 = context.WithValue(ctx2, key, ctx.Value(key))

// 修复后:
ctx2 := context.WithoutCancel(ctx)


// ===== min/max Modernizers =====

// 9. minmax - 使用内置 min/max
// 修复前:
if a < b {
    min = a
} else {
    min = b
}

// 修复后:
min = min(a, b)


// ===== errors 包 Modernizers =====

// 10. asany - 使用 errors.AsType
// 修复前:
var myErr *MyError
if errors.As(err, &myErr) { ... }

// 修复后:
if myErr, ok := errors.AsType[*MyError](err); ok { ... }


// 11. joinerrors - 使用 errors.Join
// 修复前:
err = fmt.Errorf("%v; %v", err1, err2)

// 修复后:
err = errors.Join(err1, err2)
```

### 2.3 完整示例: 运行 go fix

假设我们有以下旧代码：

```go
// main.go (修复前)
package main

import (
    "fmt"
    "sort"
)

func main() {
    // 使用旧方式查找
    nums := []int{1, 2, 3, 4, 5}
    target := 3

    found := false
    for _, v := range nums {
        if v == target {
            found = true
            break
        }
    }
    fmt.Println("Found:", found)

    // 使用旧方式排序
    sort.Slice(nums, func(i, j int) bool {
        return nums[i] < nums[j]
    })

    // 使用旧方式找最小值
    a, b := 10, 20
    var min int
    if a < b {
        min = a
    } else {
        min = b
    }
    fmt.Println("Min:", min)
}
```

运行 `go fix` 后：

```go
// main.go (修复后)
package main

import (
    "fmt"
    "slices"  // 自动导入
)

func main() {
    nums := []int{1, 2, 3, 4, 5}
    target := 3

    // 自动替换为 slices.Contains
    found := slices.Contains(nums, target)
    fmt.Println("Found:", found)

    // 自动替换为 slices.Sort
    slices.Sort(nums)

    // 自动替换为内置 min
    a, b := 10, 20
    min := min(a, b)
    fmt.Println("Min:", min)
}
```

### 2.4 项目级 Modernization

```bash
# 1. 备份代码
git add -A && git commit -m "Before go fix"

# 2. 预览所有可能的修复
go fix -n ./... 2>&1 | head -100

# 3. 应用修复
go fix ./...

# 4. 整理导入
goimports -w .

# 5. 运行测试确保行为不变
go test ./...

# 6. 提交修复
git add -A && git commit -m "Apply go fix modernizers"
```

## 3. //go:fix inline 指令

### 3.1 概述

`//go:fix inline` 指令允许你**标记函数为可内联**，`go fix` 会自动将所有调用替换为函数体。

这特别适用于：

- API 弃用和迁移
- 函数重命名
- 简单的包装函数消除

### 3.2 基本语法

```go
//go:fix inline
func OldFunction() {
    NewFunction()
}

func NewFunction() {
    // 新实现
}
```

### 3.3 使用场景

#### 场景1: API 重命名

```go
package myapi

//go:fix inline
// Deprecated: Use NewProcess instead.
func ProcessData(data []byte) error {
    return NewProcess(data)
}

func NewProcess(data []byte) error {
    // 新的处理逻辑
    return nil
}
```

运行 `go fix` 后，调用者代码：

```go
// 用户代码 (修复前)
err := myapi.ProcessData(data)

// 用户代码 (修复后)
err := myapi.NewProcess(data)
```

#### 场景2: 参数变更

```go
package config

//go:fix inline
// Deprecated: Use LoadWithOptions instead.
func Load(path string) (*Config, error) {
    return LoadWithOptions(path, DefaultOptions())
}

func LoadWithOptions(path string, opts Options) (*Config, error) {
    // 新实现
    return nil, nil
}
```

#### 场景3: 简化包装

```go
package utils

//go:fix inline
func GetString(m map[string]any, key string) string {
    return GetStringDefault(m, key, "")
}

func GetStringDefault(m map[string]any, key, def string) string {
    if v, ok := m[key].(string); ok {
        return v
    }
    return def
}
```

### 3.4 内联规则

```go
// ✅ 可以内联: 简单转发
//go:fix inline
func A() { B() }

// ✅ 可以内联: 参数传递
//go:fix inline
func A(x int) { B(x + 1) }

// ✅ 可以内联: 结果返回
//go:fix inline
func A() int { return B() }

// ✅ 可以内联: 选择和转换
//go:fix inline
func A(x int) int {
    if x > 0 {
        return B(x)
    }
    return C(x)
}

// ❌ 不能内联: 有副作用的语句
//go:fix inline
func A() {
    println("side effect")  // 副作用
    B()
}

// ❌ 不能内联: 复杂控制流
//go:fix inline
func A() {
    for i := 0; i < 10; i++ {  // 循环
        B()
    }
}

// ❌ 不能内联: 返回值不匹配
//go:fix inline
func A() (int, error) {
    B()  // B 返回不同值
    return 0, nil
}
```

### 3.5 完整迁移示例

```go
// v1/api.go
package v1

import "example.com/mymod/v2"

//go:fix inline
// Deprecated: Use v2.Process instead.
func Process(data []byte) error {
    return v2.Process(data)
}

//go:fix inline
// Deprecated: Use v2.Config instead.
type Config = v2.Config

//go:fix inline
// Deprecated: Use v2.DefaultTimeout instead.
const DefaultTimeout = v2.DefaultTimeout
```

用户代码迁移：

```bash
# 1. 用户更新依赖
go get example.com/mymod/v2

# 2. 运行 go fix 自动迁移
cd userproject
go fix ./...

# 3. 手动更新导入 (如果需要)
goimports -w .
```

## 4. 其他工具改进

### 4.1 go mod init 默认版本

```bash
# Go 1.26 工具链
go version
go version go1.26.0 linux/amd64

# go mod init 默认使用较低的 Go 版本
go mod init example.com/mymod

# cat go.mod
module example.com/mymod

go 1.25.0  # 使用 N-1 版本，而非 1.26

# 显式设置版本
go mod edit -go=1.26
```

### 4.2 cmd/doc 移除

```bash
# ❌ 已移除
go tool doc

# ✅ 使用 go doc 替代
go doc fmt.Printf
go doc net/http
```

### 4.3 编译器标志

```bash
# 新的切片栈分配优化
# 如果遇到问题，可以禁用
go build -gcflags=all=-d=variablemakehash=n ./...

# 使用 bisect 工具定位问题
# go install golang.org/x/tools/cmd/bisect@latest
bisect -compile=variablemake go test ./...
```

### 4.4 自定义 Modernizer 开发

```go
// mymodernizer.go
package mymodernizer

import (
    "golang.org/x/tools/go/analysis"
)

var Analyzer = &analysis.Analyzer{
    Name: "mymodernizer",
    Doc:  "Replace OldFunc with NewFunc",
    Run:  run,
}

func run(pass *analysis.Pass) (interface{}, error) {
    // 实现分析逻辑
    // ...

    // 报告问题并建议修复
    pass.Report(analysis.Diagnostic{
        Pos:     call.Pos(),
        Message: "should use NewFunc instead of OldFunc",
        SuggestedFixes: []analysis.SuggestedFix{{
            Message: "Replace with NewFunc",
            TextEdits: []analysis.TextEdit{{
                Pos:     call.Fun.Pos(),
                End:     call.Fun.End(),
                NewText: []byte("NewFunc"),
            }},
        }},
    })

    return nil, nil
}
```
