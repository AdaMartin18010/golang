# F1: 元语言与符号系统

> **层级**: 基础层 (Foundation)
> **地位**: 整个知识体系的元语言基础
> **依赖**: 无（这是基础）

---

## 1. 逻辑符号

### 1.1 量词

| 符号 | 读法 | 含义 | 示例 |
|------|------|------|------|
| `∀` | 对于所有 | 全称量词 | `∀x∈S. P(x)` 表示S中所有x满足P |
| `∃` | 存在 | 存在量词 | `∃x. P(x)` 表示存在x满足P |
| `∃!` | 存在唯一 | 唯一存在 | `∃!x. P(x)` 表示唯一的x满足P |

### 1.2 逻辑连接词

| 符号 | 名称 | 真值表 | Go对应 |
|------|------|--------|--------|
| `¬` | 非 | ¬T=F, ¬F=T | `!` |
| `∧` | 与 | T∧T=T, 其他F | `&&` |
| `∨` | 或 | F∨F=F, 其他T | `\|\|` |
| `→` | 蕴含 | F→T=T, T→F=F | 无直接对应 |
| `↔` | 等价 | 两边同值时为T | `==` |

### 1.3 推理符号

| 符号 | 含义 | 使用场景 |
|------|------|----------|
| `⊢` | 推导/证明 | `Γ ⊢ e: T` 表示在上下文Γ中e具有类型T |
| `⊨` | 语义蕴涵 | `Γ ⊨ e: T` 表示语义上成立 |
| `≡` | 定义等价 | `A ≡ B` 表示A定义为等价于B |
| `≜` | 定义为 | `f(x) ≜ x+1` 表示f定义为x+1 |

## 2. 类型理论符号

### 2.1 基本类型构造

```
Type ::=                     (类型)
    | Base                   (基础类型)
    | Type → Type            (函数类型)
    | Type × Type            (积类型/元组)
    | Type + Type            (和类型/联合)
    | μX.Type                (递归类型)
    | ∀X.Type                (全称类型/泛型)
    | *Type                  (指针类型)

Base ::=
    | int | int8 | int16 | int32 | int64
    | uint | uint8 | uint16 | uint32 | uint64
    | float32 | float64
    | bool | string
```

### 2.2 类型判断

```
类型判断形式: Γ ⊢ e : T

其中:
  Γ: 类型环境 (Type Environment)
     Γ ::= ∅ | Γ, x:T
     表示变量到类型的映射

  e: 表达式 (Expression)

  T: 类型 (Type)

示例:
  x:int, y:bool ⊢ x + 1 : int
  表示: 在x是int、y是bool的环境中，x+1具有类型int
```

## 3. 操作语义符号

### 3.1 小步语义

```
形式: e → e'

含义: 表达式e单步规约为e'

规则书写格式:
  前提1   前提2   ...   前提n
  ──────────────────────────────  (规则名)
         结论

示例:
  e₁ → e₁'
  ─────────────────  (Congruence-Add)
  e₁ + e₂ → e₁' + e₂
```

### 3.2 大步语义

```
形式: e ⇓ v

含义: 表达式e求值到值v

示例:
  3 ⇓ 3                (值已经是值)
  1+2 ⇓ 3              (求值结果)
```

### 3.3 求值上下文

```
E ::=                      (求值上下文)
    | []                   (空上下文/洞)
    | E + e | v + E        (加法上下文)
    | E(e) | v(E)          (应用上下文)
    | *E | &E              (指针上下文)

使用:
  E[e] 表示将e填入上下文E的洞中

示例:
  如果 E = [] + 2，那么 E[1] = 1 + 2
```

## 4. 指称语义符号

### 4.1 语义函数

```
⟦·⟧ : Expression → Environment → Value

含义: 将表达式映射到其在某环境下的值

示例:
  ⟦x⟧ρ = ρ(x)          (变量在环境中查找)
  ⟦n⟧ρ = n             (常量映射到自身)
  ⟦e₁+e₂⟧ρ = ⟦e₁⟧ρ + ⟦e₂⟧ρ  (复合表达式)
```

### 4.2 环境

```
Environment ρ : Variable → Value

操作:
  ρ[x ↦ v] : 将x映射到v的新环境

示例:
  设 ρ = {x ↦ 3, y ↦ true}
  则 ρ[x ↦ 5] = {x ↦ 5, y ↦ true}
```

## 5. Go特定符号

### 5.1 内存模型

```
State σ : Address → Value

操作:
  alloc(σ, n) = (p, σ')   分配n字节，返回指针p和新状态σ'
  load(σ, p) = v          从地址p加载值v
  store(σ, p, v) = σ'     将v存储到p，返回新状态

指针: p ∈ Address
地址运算: &x 表示x的地址，*p 表示p指向的值
```

### 5.2 泛型符号

```
Type Parameter: X, Y, Z, ...

泛型实例化:
  instantiate(F, T) = F[T/X]  (将F中的X替换为T)

约束:
  C : Type → Constraint
  T satisfies C ↔ C(T) holds
```

## 6. 证明符号

### 6.1 证明结构

```
定理: P → Q

证明:
  1. 假设P
  2. ... (推导步骤)
  n. 得到Q
  ∴ P → Q 成立

符号:
  ∴ : 因此/所以
  □ : 证毕 (QED)
```

### 6.2 归纳法

```
结构归纳:
  要证明: ∀x∈S. P(x)

  基础情况: 证明P(基础元素)
  归纳步骤: 假设P(x)，证明P(构造(x))

数学归纳:
  要证明: ∀n∈ℕ. P(n)

  基础: 证明P(0)
  归纳: 假设P(k)，证明P(k+1)
```

## 7. 约定与规范

### 7.1 命名规范

| 类别 | 命名 | 示例 |
|------|------|------|
| 类型 | T, U, S | `T: Type` |
| 值 | v, u, w | `v: T` |
| 表达式 | e, f, g | `e: Expression` |
| 环境 | Γ, Δ | `Γ: Environment` |
| 状态 | σ, σ' | `σ: State` |
| 地址 | p, q | `p: Address` |

### 7.2 排版规范

```
定义块:
┌─────────────────────────────┐
│ 定义名: 内容                 │
│ 含义: 解释                   │
└─────────────────────────────┘

定理块:
┌─────────────────────────────┐
│ 定理: 陈述                   │
│ 证明: ...                    │
│ ∴ 证毕                       │
└─────────────────────────────┘
```
