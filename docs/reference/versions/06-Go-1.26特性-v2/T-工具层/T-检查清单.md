```markdown

- [ ] 修改 go.mod: `go 1.26`
```

## 二、代码审查清单

### 2.1 new表达式使用

```markdown
## new表达式检查
- [ ] 是否用于可选字段？new(v) 比 &[]T{v}[0] 更清晰
- [ ] 是否用于延迟初始化？
- [ ] 是否考虑逃逸分析？小对象可能栈分配
- [ ] 是否保持代码风格一致？
```

### 2.2 递归泛型使用

```markdown
## 递归泛型检查
- [ ] 约束是否结构递归？（确保终止）
- [ ] Children() 方法是否返回子结构？
- [ ] 是否有基础情况？（防止无限递归）
- [ ] 类型参数使用是否清晰？
```

## 三、性能优化清单

### 3.1 GC优化

```markdown
## GC优化检查
- [ ] GOGC设置是否合理？（默认100）
- [ ] GOMEMLIMIT是否设置？（容器环境）
- [ ] 是否使用sync.Pool？
- [ ] 是否避免热路径分配？
- [ ] 是否监控GC暂停？
```

### 3.2 SIMD利用

```markdown
## SIMD优化检查
- [ ] 是否使用标准库的bytes/strings函数？
- [ ] 数据量是否足够大（>64B）？
- [ ] 是否批量处理数据？
- [ ] 是否避免过早优化？
```

## 四、安全检查清单

### 4.1 HPKE使用

```markdown
## HPKE安全清单
- [ ] 使用推荐的算法套件？
  - KEM: DHKEM_X25519_HKDF_SHA256
  - KDF: HKDF_SHA256
  - AEAD: AES_256_GCM 或 ChaCha20Poly1305
- [ ] 是否使用Context信息绑定？
- [ ] 密钥是否安全存储？
- [ ] 是否实现密钥轮换？
```

### 4.2 敏感数据处理

```markdown
## 敏感数据检查
- [ ] 是否使用crypto/subtle.ZeroBytes？
- [ ] 密钥是否在内存中及时清除？
- [ ] 是否避免日志记录敏感信息？
- [ ] 是否使用defer确保清除？
```
