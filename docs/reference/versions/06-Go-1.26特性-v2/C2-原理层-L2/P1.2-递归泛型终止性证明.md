# 证明 P1.2: 递归泛型终止性

> **文档层级**: C2-原理层 (Principle Layer L2)
> **文档类型**: 形式证明 (Formal Proof)
> **定理**: [Th1.2](../R-参考层/R-定理索引.md#Th1.2)

## 一、定理陈述

```
Th1.2: ∀C[T C[T]]: Constraint. wellformed(C) → terminates(unfold(C))

自然语言:
对于任意结构递归良好的递归类型约束C[T C[T]]，
约束展开过程在有限步内终止。
```

---

## 二、预备知识

### 2.1 结构递归定义

```
定义: 结构递归 (Structural Recursion)

一个递归定义是结构递归的，如果:
1. 每次递归调用时，参数是原结构的"子结构"
2. 子结构严格小于原结构（某种度量下）
3. 存在基础情况（递归停止条件）

示例:
  - 树的子树比原树小
  - 链表的尾比原链表短
  - AST的子节点比父节点简单
```

### 2.2 良基关系

```
定义: 良基关系 (Well-founded Relation)

关系 < 在集合S上是良基的，如果不存在无限降链:
  ¬∃s₁, s₂, s₃, ... ∈ S. s₁ > s₂ > s₃ > ...

性质:
  - 每个非空子集有极小元
  - 支持数学归纳法
```

## 三、完整证明

### 3.1 证明策略

```
策略: 结构归纳法 (Structural Induction)

步骤:
1. 定义类型结构的度量（深度/大小）
2. 证明展开操作减小度量
3. 应用良基归纳
4. 得出终止结论
```

### 3.2 证明

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
证明:

定义 1 (类型深度):
  depth(T) =
    0                         如果T是基础类型
    1 + max(depth(Fᵢ))        如果T是struct{Fᵢ}
    1 + depth(E)              如果T是[]E或*E
    1 + max(depth(A), depth(R)) 如果T是func(A)R

定义 2 (约束展开):
  unfold(C[T], n) = 第n步展开后的约束

定义 3 (结构良好):
  wellformed(C) ↔ C的递归引用使类型深度严格递减

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
引理 1: 结构递归保证深度递减

命题:
  如果C是结构递归的，那么:
  ∀T. depth(unfold(C[T], n+1)) < depth(unfold(C[T], n))

证明:
  结构递归定义要求递归引用时使用"子结构"。

  对于树遍历示例:
    C[Tree] = interface { Children() []Tree }

    在Children()中返回的子树严格小于父树
    ∴ depth(child) < depth(parent)

  一般地，结构递归确保:
    unfold(C[T], n+1) 引用 C[T'] 其中 depth(T') < depth(T)

  ∴ depth(unfold(C[T], n+1)) < depth(unfold(C[T], n))

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
引理 2: Go类型系统的良基性

命题:
  Go类型系统是良基的（不存在无限降链）

证明:
  Go类型由有限的基本构造组成:
    - 基础类型: int, string, bool, ...
    - 复合类型: struct, array, slice, map, func, interface, chan

  每个复合类型的定义引用其他类型，
  但引用链必须是有限的（Go语言规定）。

  因此，类型深度函数depth是良基的。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
定理证明 (主证明):

假设:
  C是结构良好的递归约束

目标:
  证明unfold(C)在有限步终止

证明:
  1. 由引理1，结构良好 ⟹ 每次展开深度递减

  2. 设d₀ = depth(T₀)是初始类型深度（有限）

  3. 展开序列:
       unfold(C[T₀], 0) = C[T₀]           深度 = d₀
       unfold(C[T₀], 1) = C[T₁]           深度 = d₁ < d₀
       unfold(C[T₀], 2) = C[T₂]           深度 = d₂ < d₁
       ...

  4. 得到递减序列: d₀ > d₁ > d₂ > ...

  5. 由引理2，depth是良基的
     ⟹ 不存在无限递减序列

  6. 因此序列必须在有限步后终止
     ⟹ ∃n. unfold(C[T₀], n) 达到不动点

  7. 达到不动点 ⟹ 终止

  ∴ terminates(unfold(C))

证毕 ∎
```

## 四、复杂度分析

### 4.1 时间复杂度

```
定理: 终止步数 ≤ 初始深度

证明:
  每次展开深度减1，初始深度为d，
  最多d步后深度为0，达到基础情况。

典型值:
  - 链表: O(n) 其中n是链表长度
  - 树: O(h) 其中h是树高
  - AST: O(d) 其中d是语法树深度
```

### 4.2 空间复杂度

```
空间: O(d) 用于存储展开过程中的中间约束

优化:
  - 可以边展开边检查，无需存储所有中间状态
  - 实际空间复杂度可优化到O(1)（递归栈空间）
```

## 五、反例分析

### 5.1 非终止的约束

```
❌ 错误示例: 非结构递归

type Bad[T Bad[Bad[T]]] interface {  // 双重嵌套
    Method(T)
}

问题:
  - 每次递归引用Bad[Bad[T]]，类型深度增加
  - 不满足结构递归条件
  - 展开不会终止

Go 1.26处理:
  编译器检测并报错: "类型约束展开不终止"
```

### 5.2 对比正确示例

```
✅ 正确示例: 结构递归

type Node[T Node[T]] interface {
    Children() []T
}

分析:
  - Children()返回的是子节点
  - 子节点在树结构中小于父节点
  - 深度递减，满足结构递归
  - 展开终止
```

## 六、实际意义

```
为什么终止性重要?

1. 编译器不会无限循环
   - 类型检查保证在合理时间内完成
   - 用户体验友好

2. 类型系统一致性
   - 所有有效程序都有有限类型推导
   - 保持 decidability

3. 运行时安全
   - 递归泛型函数不会无限递归（由结构保证）
```
