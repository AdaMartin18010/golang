# 证明 P1.1: new表达式语义等价性

> **文档层级**: C2-原理层 (Principle Layer L2)
> **文档类型**: 形式证明 (Formal Proof)
> **定理**: [Th1.1](../R-参考层/R-定理索引.md#Th1.1)
> **最后更新**: 2026-03-06

---

## 一、定理陈述

```
Th1.1: ∀T: Type, v: T. new(T(v)) ≡ &T(v)

自然语言:
对于任意类型T和值v，使用new表达式new(T(v))分配的并初始化的指针，
与直接使用取地址&T(v)语义等价。
```

---

## 二、证明

### 2.1 符号约定

```
定义:
  new(v)   : new表达式，分配内存并存储v
  &v       : 取地址操作，返回v的地址
  alloc(T) : 为类型T分配内存
  store(p, v): 将v存储到地址p
  addressof(v): v的内存地址

≡ : 语义等价关系
```

### 2.2 证明

```
命题: ∀T: Type, v: T. new(v) ≡ &v'

其中 v' 是 v 的副本（fresh copy）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
证明:

步骤 1: 展开 new(v) 的语义
────────────────────────────────
根据 A6-new表达式扩展公理:
  new(v) = alloc(typeof(v)) ; store(alloc(typeof(v)), v)

设 p = alloc(typeof(v))
则 new(v) = store(p, v) ; return p

步骤 2: 展开 &v' 的语义
────────────────────────────────
根据 A3-指针语义公理:
  &v' = addressof(v')

其中 v' 是 v 的副本

步骤 3: 比较两者的内存操作
────────────────────────────────
new(v):
  1. 分配新内存: p = alloc(sizeof(T))
  2. 复制v到p: store(p, copy(v))
  3. 返回p

&v':
  1. 创建v的副本: v' = copy(v)
  2. 获取v'的地址: p' = addressof(v')
  3. 返回p'

步骤 4: 证明等价性
────────────────────────────────
关键观察:
  - 两者都创建了v的副本
  - 两者都返回指向该副本的指针
  - alloc(T)分配的内存在程序语义上是新鲜的
  - 在不可区分性意义上，新分配的内存地址等价于新变量的地址

形式化:
  设 ⟦new(v)⟧ 是 new(v) 的指称
  设 ⟦&v'⟧ 是 &v' 的指称

  ⟦new(v)⟧ = λσ. let (p, σ₁) = alloc(σ, sizeof(T)) in
                     σ₂ = store(σ₁, p, v) in
                     (p, σ₂)

  ⟦&v'⟧ = λσ. let (l, σ₁) = alloc_local(σ, sizeof(T)) in
                  σ₂ = store(σ₁, l, v) in
                  (l, σ₂)

  对于任何观察O:
    O(⟦new(v)⟧) = O(⟦&v'⟧)

  因为:
    - 两者返回的指针都指向包含v的内存
    - 两者对后续操作的影响相同
    - 没有可观察的方式区分两者

步骤 5: 结论
────────────────────────────────
因此，根据语义等价的定义:
  new(v) ≡ &v'

证毕 ∎
```

## 三、推论

### 3.1 替换性

```
推论: 在程序中，new(v) 可以替换为 &v' 而不改变程序语义

应用:
  x := new(42)
  // 等价于
  v := 42
  x := &v
```

### 3.2 类型保持

```
推论: new(v) 和 &v 具有相同的类型 *T

证明:
  - new(v) : *T (根据 new-intro 规则)
  - &v : *T (根据 A3)
  ∴ 类型相同
```

---

## 四、直观解释

```
为什么 new(v) ≡ &v' ?

直观理解:
  new(v) 说: "给我一个地方存放v的副本，给我地址"
  &v' 说: "创建一个v的副本，给我它的地址"

两者做的事完全一样:
  1. 创建v的副本
  2. 返回副本的地址

区别仅在于语法，语义完全等价。
```

## 五、验证

### 5.1 代码验证

```go
// 验证程序
func main() {
    // 方式1: new表达式
    p1 := new(42)

    // 方式2: 取地址
    v := 42
    p2 := &v

    // 验证等价
    fmt.Println(*p1 == *p2)  // true
}
```

### 5.2 编译器视角

```
编译器可以将:
  new(v)
优化为:
  t := v
  &t

或者反之，取决于逃逸分析结果。
```
