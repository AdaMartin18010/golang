## 一、形式语义定义

### 1.1 操作语义

```
表达式: new(e)

求值规则:
────────────────────────────────
Γ ⊢ e ↓ v               (e求值为v)
Γ ⊢ alloc(typeof(v)) ↓ p    (分配内存得地址p)
Γ ⊢ store(p, v) ↓ ()      (存储v到p)
────────────────────────────────
Γ ⊢ new(e) ↓ p            (new(e)求值为p)
```

### 1.2 小步语义

```
new(e) → let p = alloc(typeof(e)) in store(p, e); p

展开:
1. e → e'                    (求值e)
2. new(e) → new(e')          (传播求值)
3. new(v) → alloc(typeof(v)) → p
4. new(v) → store(p, v)
5. new(v) → p                (最终值)
```

### 1.3 指称语义

```
⟦new(e)⟧ρ =
  let v = ⟦e⟧ρ in
  let p = alloc(typeof(v)) in
  store(p, v);
  p

其中:
  ρ: 环境（变量到值的映射）
  alloc: 内存分配函数
  store: 存储函数
```

## 二、类型系统

### 2.1 类型规则

```
[new-intro]
────────────────────────────────
Γ ⊢ e : T    T is value type
────────────────────────────────
Γ ⊢ new(e) : *T

[new-elim]
────────────────────────────────
Γ ⊢ p : *T
────────────────────────────────
Γ ⊢ *p : T

[new-congruence]
────────────────────────────────
Γ ⊢ e₁ ≡ e₂
────────────────────────────────
Γ ⊢ new(e₁) ≡ new(e₂)
```

### 2.2 类型安全性

**定理 (Type Preservation)**:

```
如果 Γ ⊢ e : T 且 e → e'，那么 Γ ⊢ e' : T

推论: new表达式保持类型
  Γ ⊢ new(v) : *T 蕴含 alloc(T) 返回 *T
```

**定理 (Progress)**:

```
如果 Γ ⊢ e : T，那么 e 是值，或存在 e' 使得 e → e'

对于new表达式:
  new(v) 是值（地址）
  new(e) 当e不是值时可继续求值
```

## 三、等价关系

### 3.1 语义等价

**定义**:

```
两个表达式语义等价 (≡) 当且仅当:
  1. 它们求值到相同的值，或
  2. 它们产生相同的可观察行为
```

**定理 Th1.1 (完整证明)**:

```
∀T: Type, v: T. new(v) ≡ &v'

证明:
  设 v : T

  左边: new(v)
    = alloc(T)                    [定义]
    ; store(alloc(T), v)          [定义]
    ; return(addr)                [定义]
    = p 其中 p 指向存储v的内存

  右边: &v'
    设 v' 是 v 的副本
    = addressof(v')               [A3]
    = p' 其中 p' 是 v' 的地址

  关键观察:
    - alloc(T) 分配新内存
    - store(p, v) 将v复制到新内存
    - v' 也是v的副本
    - 因此 p 和 p' 都指向v的等价副本
    - 在不可区分性意义上，p ≡ p'

  ∴ new(v) ≡ &v'
```

### 3.2 上下文等价

```
定义: e₁ ≅ e₂ 当且仅当对于所有上下文C，C[e₁] ≡ C[e₂]

性质:
  new(v₁) ≅ new(v₂)  当 v₁ ≡ v₂

  这意味着:
    x := new(42)
    // 等价于
    x := &[]int{42}[0]
```

## 四、内存模型

### 4.1 分配语义

```
状态: σ = (H, S) 其中 H是堆，S是栈

分配规则:
  (H, S), alloc(T) → (H', S), p
  其中:
    p ∉ dom(H)        (新地址)
    H' = H[p ↦ uninit(T)]  (未初始化内存)
    size(uninit(T)) = sizeof(T)

存储规则:
  (H, S), store(p, v) → (H', S), ()
  其中:
    H' = H[p ↦ v]     (存储值)
```

### 4.2 逃逸分析形式化

```
定义: escapes(x) 当且仅当变量x的地址超出其作用域被使用

规则:
  escapes(x) if:
    1. &x 被返回
    2. &x 被赋值给全局变量
    3. &x 被传递给未知函数
    4. &x 被发送到channel

优化:
  if ¬escapes(x) then:
    alloc(x) → stack_alloc(x)
  else:
    alloc(x) → heap_alloc(x)
```

## 五、推理示例

### 5.1 程序验证

```go
// 程序
func f() *int {
    return new(42)
}

// 验证目标: 返回值是指向42的指针
// 证明:
//   1. new(42) 根据[new-intro]有类型 *int
//   2. 根据Th1.1，new(42) ≡ &42'
//   3. 函数返回 *int 类型值，类型正确
//   4. 返回的指针指向包含42的内存
```

### 5.2 等价转换

```go
// 原始代码
ptr := &[]int{compute()}[0]

// 等价转换 (Th1.1)
ptr := new(compute())

// 验证等价性:
//   - 两者都分配内存存储compute()的结果
//   - 两者都返回指向该内存的指针
//   - 根据Th1.1，语义等价
```
