## 一、模式概述

### 1.1 问题场景

需要实现通用的树遍历算法，支持不同类型的树结构：

```go
// 文件系统树
type FileNode struct {
    Name     string
    Children []*FileNode
}

// AST节点
type ASTNode struct {
    Type     string
    Children []*ASTNode
}

// 问题：为每种树写重复的遍历代码？
```

### 1.2 解决方案

使用Go 1.26递归泛型约束抽象通用树遍历：

```go
// 通用树节点约束
type Node[T Node[T]] interface {
    Children() []T
}

// 通用遍历函数
func Walk[T Node[T]](node T, fn func(T)) {
    fn(node)
    for _, child := range node.Children() {
        Walk(child, fn)  // 递归调用，类型安全
    }
}
```

## 二、模式实现

### 2.1 基础实现

```go
package tree

// Node 树节点接口
type Node[T Node[T]] interface {
    Children() []T
}

// Walk 前序遍历
func Walk[T Node[T]](node T, fn func(T)) {
    fn(node)
    for _, child := range node.Children() {
        Walk(child, fn)
    }
}

// WalkPostOrder 后序遍历
func WalkPostOrder[T Node[T]](node T, fn func(T)) {
    for _, child := range node.Children() {
        WalkPostOrder(child, fn)
    }
    fn(node)
}

// Find 查找满足条件的节点
func Find[T Node[T]](node T, predicate func(T) bool) (T, bool) {
    var zero T
    if predicate(node) {
        return node, true
    }
    for _, child := range node.Children() {
        if found, ok := Find(child, predicate); ok {
            return found, true
        }
    }
    return zero, false
}

// Collect 收集所有满足条件的节点
func Collect[T Node[T]](node T, predicate func(T) bool) []T {
    var result []T
    Walk(node, func(n T) {
        if predicate(n) {
            result = append(result, n)
        }
    })
    return result
}
```

### 2.2 具体应用

```go
// 文件系统节点
type FileNode struct {
    Name     string
    Size     int64
    IsDir    bool
    Children []*FileNode
}

func (f *FileNode) Children() []*FileNode {
    return f.Children
}

// 使用遍历
func main() {
    root := buildFileTree("/home/user")

    // 打印所有文件
    tree.Walk(root, func(node *FileNode) {
        if !node.IsDir {
            fmt.Printf("%s: %d bytes\n", node.Name, node.Size)
        }
    })

    // 查找大文件
    bigFiles := tree.Collect(root, func(node *FileNode) bool {
        return node.Size > 100*1024*1024 // >100MB
    })
}
```

## 三、进阶模式

### 3.1 访问者模式

```go
// Visitor 访问者接口
type Visitor[T Node[T]] interface {
    Visit(node T)
}

// Accept 接受访问者
func Accept[T Node[T]](node T, visitor Visitor[T]) {
    Walk(node, visitor.Visit)
}

// 具体访问者：统计节点
type Counter[T Node[T]] struct {
    Count int
}

func (c *Counter[T]) Visit(node T) {
    c.Count++
}

// 使用
func main() {
    root := buildTree()
    counter := &Counter[*FileNode]{}
    tree.Accept(root, counter)
    fmt.Printf("Total nodes: %d\n", counter.Count)
}
```

### 3.2 带上下文遍历

```go
// WalkWithContext 带上下文的遍历
func WalkWithContext[T Node[T], C any](
    node T,
    ctx C,
    fn func(T, C) C,
) {
    newCtx := fn(node, ctx)
    for _, child := range node.Children() {
        WalkWithContext(child, newCtx, fn)
    }
}

// 示例：计算深度
type DepthContext struct {
    Depth int
    Path  string
}

func printWithDepth(node *FileNode, ctx DepthContext) DepthContext {
    indent := strings.Repeat("  ", ctx.Depth)
    fmt.Printf("%s%s\n", indent, node.Name)
    return DepthContext{
        Depth: ctx.Depth + 1,
        Path:  ctx.Path + "/" + node.Name,
    }
}
```

## 四、性能考量

### 4.1 时间复杂度

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| Walk | O(n) | n为节点数 |
| Find | O(n) | 最坏遍历所有节点 |
| Collect | O(n) | 遍历+收集 |

### 4.2 空间复杂度

```
空间: O(h) 递归栈深度，h为树高

优化:
  - 对于深树，考虑使用显式栈迭代实现
  - 或增加递归深度限制
```

## 五、最佳实践

### 5.1 设计建议

```go
// ✅ 推荐：简洁的Children方法
func (n *Node) Children() []*Node {
    return n.Children
}

// ✅ 推荐：使用指针切片（避免复制）
func (n *Node) Children() []*Node { ... }

// ⚠️ 注意：确保Children返回稳定结果
// 不要在Children方法中修改树结构
```

### 5.2 错误处理

```go
// WalkWithError 支持错误返回
func WalkWithError[T Node[T]](
    node T,
    fn func(T) error,
) error {
    if err := fn(node); err != nil {
        return err
    }
    for _, child := range node.Children() {
        if err := WalkWithError(child, fn); err != nil {
            return err
        }
    }
    return nil
}
```
