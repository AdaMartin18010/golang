## 技术架构亮点

### 三层AI友好架构

```
┌─────────────────────────────────────────────────────────┐
│ AI 交互层                                               │
│ • 自然语言查询    • 智能代码生成    • 运行时诊断        │
└─────────────────────────────────────────────────────────┘
                           │
┌─────────────────────────────────────────────────────────┐
│ MCP服务层 (Model Context Protocol)                      │
│ • 标准化AI调用接口    • 支持Claude/GPT等主流LLM         │
└─────────────────────────────────────────────────────────┘
                           │
┌─────────────────────────────────────────────────────────┐
│ 知识图谱层                                              │
│ Neo4j (图存储) + Weaviate (向量检索)                    │
│ • 组件依赖关系    • 配置-指标关联    • 语义搜索         │
└─────────────────────────────────────────────────────────┘
```

### 差异化技术特色

1. **自举设计**: 使用OTel收集自身指标，验证概念
2. **运行时闭环**: 静态知识 + 运行时数据 = 智能诊断
3. **渐进式扩展**: 从单项目到生态平台，每个阶段有产出
4. **验证机制**: 知识可溯源、可验证、置信度评分

**首个面向AI的开源可观测性项目知识图谱系统**

让AI像资深工程师一样：

- 理解OpenTelemetry、Prometheus等基础设施
