### 1.2 代码问题详细分析

#### 问题 1: errors.AsType 示例

**当前代码**:

```go
// 有问题的代码
customErr2, ok := errors.AsType[*CustomError](err)
```

**问题**:

- `errors.AsType` 是真实存在的 Go 1.26 特性
- 但示例中使用的错误类型在标准库中不可导出

**修复建议**:

```go
// 正确的示例
type CustomError struct {
    Code int
}
func (e *CustomError) Error() string { return "custom" }

func main() {
    var err error = &CustomError{Code: 404}
    if ce, ok := errors.AsType[*CustomError](err); ok {
        fmt.Println(ce.Code)
    }
}
```

#### 问题 2: new() 表达式理解错误

**我的理解错误**:
我误以为 `new(int64(300))` 这样的代码可以在 Go 1.26 中工作，但实际上：

```go
// 这是错误的！
ptr := new(int64(300))  // 编译错误: 类型不是表达式
```

**正确的理解**:
Go 1.26 的 `new()` 支持的是：

```go
// 正确的用法
const x = int64(300)
ptr := new(x)  // 现在 x 可以作为 new 的参数

// 或者类型转换
ptr := new(int64)  // 一直都是可以的
*ptr = 300
```

**文档修正**:
需要修改所有提到 `new(int64(300))` 的文档内容。
